# Manual review checklist

- [ ] В шапке есть badge, title и subtitle.
- [ ] Badge — это короткая тема документа, а не тупой дубль заголовка или случайное последнее слово.
- [ ] Есть aurora background и card-стили (blur + border + radius).
- [ ] Текст читаем на мобиле (<= 820px), нет горизонтального скролла.
- [ ] Для обычного `/present` страница выглядит как единый красивый документ сверху вниз, а не как псевдо-слайды.
- [ ] Списки/таблицы выглядят аккуратно, без визуального мусора.
- [ ] Visual-pass: есть минимум 1 архитектурная схема (pipeline/flow/блок-схема).
- [ ] Visual-pass: есть минимум 1 сравнительный визуальный блок (table/compare/before-after).
- [ ] Если тема содержит process / architecture / KPI / old-vs-new, выбран не только текст, но и подходящий visual block (`@flow`, `@chart`, `@beforeafter`).
- [ ] В footer есть отметка генерации и timestamp.
- [ ] Файл отправляется в чат как вложение `.html`, не ссылкой.
- [ ] Для `/present slides` заголовки слайдов продвигают мысль, а не просто называют тему.
- [ ] Для `/present slides` нет ощущения документа, нарезанного на экраны; есть ритм и одна главная мысль на слайд.
- [ ] Для `/present slides` перегруженные слайды раздроблены на несколько экранов.
